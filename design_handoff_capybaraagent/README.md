# Handoff: CapybaraAgent — AI-ассистент (десктоп UI)

## Overview
CapybaraAgent — десктопный клиент локального AI-агента в стиле классического чат-ассистента
(LibreChat / Claude). Тёмная тема, эстетика «liquid glass» (macOS 26/27): полупрозрачная боковая
панель на всю высоту, тёплый «капибара-оранжевый» акцент, мягкие световые подложки за стеклом.

Ключевые возможности интерфейса:
- Локальная авторизация: вход и создание пользователя (имя/логин/пароль);
- Чат с агентом (welcome-экран + активный диалог со стримингом, вызовами инструментов, кодом,
  цитатами, вложениями и артефактами);
- Настройки: LLM/модели, MCP-серверы, Память, Фоновые задачи (по расписанию);
- Мастера подключения: привязка MCP-сервера и привязка LLM-провайдера (модальные wizard'ы
  форма → проверка соединения → успех).

## About the Design Files
Файлы в этом бандле — **дизайн-референс, сделанный в HTML** (прототип, показывающий внешний вид и
поведение), **а не продакшн-код для копирования**. Задача — **воссоздать эти экраны в целевой кодовой
базе** её родными средствами и паттернами (React/Vue/Svelte/SwiftUI/native и т.д.). Если окружения ещё
нет — выбрать подходящий фреймворк (рекомендуется **React + TypeScript**, т.к. прототип уже построен на
компонентной модели с состоянием) и реализовать дизайн там.

Прототип написан как единый «Design Component» (`CapybaraAgent.dc.html`) — это HTML со встроенным
рантаймом (`support.js`), который рендерит шаблон + класс-логику по модели, близкой к React-классу
(`state`, `setState`, `renderVals()` вместо `render()`). Читайте его как **источник истины по разметке,
стилям и логике**, но переносите в идиоматичные компоненты вашего стека. `support.js` — локальный рантайм
прототипа, в продакшн его переносить не нужно.

Чтобы просто посмотреть прототип: откройте `CapybaraAgent.dc.html` в браузере (рядом должен лежать
`support.js`).

## App flow / точки входа
Приложение **стартует с экрана авторизации** (`view:'auth'`). После входа/регистрации —
основной интерфейс (`view` становится `'chat'`, welcome). Выход из профиля (меню на карточке
пользователя внизу сайдбара) возвращает на экран входа. Пока `view==='auth'` — сайдбар и main
не монтируются (виден только центрированный экран авторизации над фоновой подложкой).

## Global Layout (основное приложение)
Полноэкранное десктопное приложение (`position:fixed; inset:0`), горизонтальный флекс:

```
┌───────────────┬──────────────────────────────────────────────┬───────────────┐
│  SIDEBAR      │  MAIN (chat ИЛИ settings)                     │  ARTIFACT     │
│  284px, glass │  flex:1                                        │  panel 420px  │
│  full height  │                                               │  (условно)    │
└───────────────┴──────────────────────────────────────────────┴───────────────┘
       Exec-detail modal — оверлей поверх всего (z-index:20)
```

За всем контентом — фиксированный фон: базовый цвет `#0d0c0b` + два больших размытых радиальных
«свечения» (warm сверху-слева, cool снизу-справа), которые медленно дрейфуют; поверх — очень слабый
шум (SVG-noise, opacity ~0.035, mix-blend overlay). Именно они создают эффект стекла на панелях.

Панель считывает фон через `backdrop-filter: blur() saturate()` — при реализации на вебе используйте
`backdrop-filter` (+ `-webkit-`), в нативе — эквивалент blur-материала.

## Interactions & Behavior
- **Авторизация**: старт на экране входа; «Войти»/«Создать аккаунт» → приложение; ссылки внизу
  карточки переключают login↔register; карточка пользователя в сайдбаре открывает popover → «Выйти
  из профиля» → экран входа.
- **Мастера подключения**: «Подключить» (MCP) / «Добавить провайдера» (LLM) открывают модальный
  wizard; «Подключить» внутри → экран проверки (лоадер) → экран успеха; «Готово»/клик по фону/X
  закрывают. В LLM-форме выбор провайдера раскрывает соответствующие поля (ключ / base URL).
- **Навигация**: «Новый чат» → welcome (очищает диалог); клик по чату → активный тред; «Настройки/
  Память/Фоновые задачи» в сайдбаре → соответствующая вкладка настроек; «‹ Назад в чат» → чат. При любой
  навигации закрывается модалка деталей.
- **Отправка сообщения**: Enter (без Shift) или кнопка send. Добавляется user-пузырь + пустой
  assistant-пузырь со `streaming:true`; заготовленный ответ печатается по 2 символа каждые 16ms;
  по завершении `streaming:false`, карет исчезает. (В проде — заменить на реальный стрим токенов.)
- **Артефакт**: клик по карточке артефакта или кнопке-тоглу в хедере открывает/закрывает правую панель;
  сегмент «Код/Превью» переключает содержимое.
- **Задачи**: клик по шапке — раскрытие/сворачивание (`expanded` map по id, шеврон 180°); клик по
  карточке-запуску — модалка деталей; клик по фону/X — закрытие.
- **Подсказки welcome**: клик подставляет текст в input.
- **Hover-состояния**: у кнопок сайдбара/чипов/карточек — осветление фона (значения в описании экранов).
- **Анимации/keyframes**: `drift`/`drift2` (дрейф свечений фона, 24s/28s), `blink` (карет, 1s steps),
  `pulse` (точка активной задачи, 2.4s), `spin` (лоадер проверки соединения в мастерах, 0.8s linear). Переходы фонов/трансформов
  — 0.12–0.2s ease. При `reduced motion` — фоновые анимации ставятся на паузу.

## State Management
Состояние прототипа (перенести в стор/локальный стейт компонента):
- `view`: `'auth' | 'chat' | 'settings'` (стартовое — `'auth'`)
- `authMode`: `'login' | 'register'`
- `userMenu`: boolean (popover выхода)
- `wizardType`: `null | 'mcp' | 'llm'`, `wizardStep`: `0` (форма) | `1` (успех), `wizConnecting`: boolean
- `wiz`: `{ name, transport:'stdio'|'http', url, provider:''|'openai'|'openrouter'|'ollama'|'custom',
  key, base }` — поля мастеров
- `chatMode`: `'welcome' | 'active'`
- `settingsTab`: `'llm' | 'mcp' | 'memory' | 'tasks'`
- `artifactOpen`: boolean, `artifactTab`: `'code' | 'preview'`
- `expanded`: `{ [taskId]: boolean }`
- `execDetail`: `{ task, run } | null`
- `appended`: массив сообщений `{ role:'user'|'assistant', text, streaming? }`
- `input`: строка ввода

Переходы авторизации: `login()/register()` → `view:'chat'` (welcome); `logout()` → `view:'auth'`
(+ сброс диалога/модалок). Мастера: `openWizard(type)` открывает форму; `wizardSubmit()` включает
`wizConnecting` и по таймеру переводит на `wizardStep:1`; `closeWizard()` закрывает. Остальные
триггеры переходов — в разделе Interactions. Данные (чаты, провайдеры, MCP-серверы, память, задачи,
запуски) в прототипе захардкожены в методах `*Data()` — в проде заменить на реальные запросы/сторы.
